# Reality Quest - Hackathon 資料

## 1. プロジェクト概要

**Reality Quest** は、AR と Live API を組み合わせた「現実探索型アドベンチャーゲーム」です。
ゲーム内で指定された条件を満たす現実アイテムを、プレイヤーが実際に探して AI に提示し、正しければ次のステージが解放されます。

- 例: 「赤くて丸いものを捧げよ」「金属で小さな鍵のようなものを示せ」
- 正解時: AR 演出 + 次の試練解放
- 不正解時: Live API が理由付きヒントを音声で返す

## 2. 何が新しいか

- AI が現実空間の物体をゲーム進行条件として扱う
- 「探す/見せる/判定される/解放される」のゲームループが即時で成立
- 同じステージでも、プレイ環境（部屋・持ち物）によって体験が変化する

## 3. 対応する問題ステートメント

**ステートメント 1: AI を活用したゲーム** に対応。

- プレイ可能でインタラクティブ
- AR + 音声対話 + 現実世界探索を融合
- AI をゲームメカニクスそのものに組み込んでいる

## 4. デモシナリオ（審査 3 分）

### 0:00 - 0:20 導入

- AR ポータルを起動
- Live API のナレーション: 「第1の試練、赤くて丸い供物を捧げよ」

### 0:20 - 1:00 現実探索 + 判定

- プレイヤーが現実のアイテムをカメラに提示
- AI 判定で正誤を返す
- 成功時に AR で「封印解除」演出

### 1:00 - 2:10 第2試練

- 条件を少し抽象化（例: 「金属で、手のひらサイズで、開閉に関係する物」）
- 不正解を1回見せ、Live API のヒント応答を実演
- 正解して次ステージへ遷移

### 2:10 - 3:00 クライマックス

- 最終試練クリア
- AR ボス/ゲート解放演出
- まとめ: 「AI が現実をゲーム化する体験」

## 5. 1 分デモ動画台本（提出用）

### 0:00 - 0:10

- タイトル + 一言: 「現実の物を探して進む AR クエスト」

### 0:10 - 0:35

- 試練提示 -> 現実アイテム提示 -> 判定成功

### 0:35 - 0:50

- 不正解例 + Live API ヒント

### 0:50 - 1:00

- 最終解放演出 + ロゴ + URL/チーム名

## 6. 技術構成（MVP）

- クライアント: WebAR (WebXR) または Unity AR Foundation
- 会話/進行: Live API（リアルタイム音声/テキスト）
- 視覚判定: 画像入力 + 条件一致スコア
- ゲーム状態管理: `quest -> validate -> unlock -> next quest`

## 7. 審査基準への対応

1. **インパクト (25%)**
   現実空間をそのままゲーム盤に変えるため、配信・イベント・教育・展示への展開性が高い。

2. **デモ (50%)**
   プレイヤーの実行動と AI のリアルタイム判定をその場で実証できる。

3. **創造性 (15%)**
   AR と Live API を「雰囲気演出」でなくコアメカニクスに使用。

4. **ピッチ (10%)**
   ルール説明が単純: 「探す -> 見せる -> 解放」。短時間で伝わる。

## 8. 失格回避のための明示事項

- 既存テンプレート利用部分と、ハッカソン中の新規実装部分を明確化
- デモでは新規実装した機能のみを中心に提示
- 禁止カテゴリ（医療助言、基本RAG、単純画像分析ツール等）に該当しないよう、ゲームループを主軸に構成

## 9. MVP スコープ（ハッカソン向け）

- 試練 3 問
- 各試練に成功/失敗演出
- 音声ヒント 2 パターン
- 最終解放シーン 1 つ

## 10. チーム内役割（例）

- AR/フロント担当: UI、空間配置、演出
- AI/バックエンド担当: Live API 接続、判定ロジック
- ストーリー/デモ担当: 試練設計、台本、動画収録
